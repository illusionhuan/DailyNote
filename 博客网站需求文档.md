# 个人静态博客网站 - 需求与架构文档


## 1. 技术栈

| 层面 | 技术 | 说明 |
|------|------|------|
| 静态站点生成器 | Hexo 8.x | Node.js 编写，Markdown 驱动，插件生态丰富，中文社区强大 |
| 主题 | Butterfly 5.5.4 | 功能最全的中文主题：内置搜索、暗色模式、卡片布局、侧边栏、TOC、代码高亮 |
| 前端搜索 | hexo-generator-searchdb | Hexo 插件，生成本地搜索索引，Butterfly 主题原生支持 |
| 内容格式 | Markdown + YAML Frontmatter | 统一写作格式，支持代码高亮、数学公式（KaTeX）、目录生成 |
| 热文抓取脚本 | Python 3 | 轻量脚本调用各平台 API，生成 JSON 数据文件 |
| 自动化构建 | GitHub Actions | 定时触发抓取脚本 + Hexo 构建 + 部署，零人工干预 |
| 部署平台 | GitHub Pages | 免费托管，支持自定义域名，与 Actions 无缝集成 |
| 版本管理 | Git + GitHub | 内容与代码统一版本控制 |
| 评论系统（可选） | Twikoo | 基于 MongoDB/Vercel，免后端，支持 Butterfly 主题集成 |

> `score` 为归一化后的 0-1000 分值，`raw_score` 为原始指标（掘金=views）。

#### 抓取脚本结构（⚠️ 待重建）

```
blog/scripts/                      # ⚠️ 目录为空，以下为设计结构
├── common.py              # 公共模块：HTTP session（含重试/SSL）、评分归一化
├── fetch_trending.py       # 主入口：抓取 → 合并去重 → 输出 JSON
├── requirements.txt        # Python 依赖：requests
└── sources/
    └── juejin.py           # 掘金 recommend_all_feed API（id_type=0）
```

**关键设计：**
- 5 个数据源通过 `ThreadPoolExecutor` 并发抓取
- 每个源独立 try/except，单源失败不影响其他
- 评分归一化：`log10(raw) × 100 × multiplier`，映射到 0-1000 分
- 合并逻辑：按 URL 去重，相同 URL 保留更高 score
- 保留历史数据：读取已有 `trending.json`，与新数据合并后取 Top 50

#### GitHub Actions 配置 (fetch_trending.yml)

已创建 `.github/workflows/fetch_trending.yml`，配置如下：

```yaml
name: Fetch Trending Articles

on:
  schedule:
    - cron: "0 0,12 * * *"    # UTC 0:00 和 12:00 = 北京时间 8:00 和 20:00
  workflow_dispatch:            # 支持手动触发

jobs:
  fetch:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.12"
      - name: Install dependencies
        run: pip install -r scripts/requirements.txt
        working-directory: blog
      - name: Fetch trending articles
        run: python scripts/fetch_trending.py
        working-directory: blog
      - name: Commit and push
        working-directory: blog
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add source/data/trending.json
          git diff --cached --quiet || git commit -m "update trending articles"
          git push
```

> `working-directory: blog` 确保脚本在 Hexo 项目根目录运行，`source/data/trending.json` 路径正确。

## 5. 快速开始命令

```bash
# 1. 安装 Node.js (>= 18.x)
# https://nodejs.org/

# 2. 安装 Hexo CLI
npm install -g hexo-cli

# 3. 创建站点
hexo init blog
cd blog

# 4. 安装 Butterfly 主题
npm install hexo-theme-butterfly --save

# 5. 安装必要插件
npm install hexo-generator-searchdb --save    # 本地搜索
npm install hexo-generator-feed --save        # RSS
npm install hexo-generator-sitemap --save     # SEO

# 6. 创建 _config.butterfly.yml（覆盖主题默认配置）

# 7. 创建第一篇文章
hexo new "我的第一篇博客"

# 8. 安装 Python 依赖（热文抓取）
pip install -r scripts/requirements.txt

# 9. 测试抓取脚本
python scripts/fetch_trending.py

# 10. 本地预览
hexo clean && hexo server

# 11. 构建部署
hexo clean && hexo generate && hexo deploy
```

## 8. 风险与注意事项

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 掘金 API 响应格式变更 | 数据缺失 | 已处理嵌套结构 `item_info.article_info`，`limit` 需为整数 |
| 掘金反爬限制 | 请求被拒 | 设置合理 User-Agent、请求间隔、仅抓取公开 API |
| GitHub Actions 额度超限 | 构建失败 | 免费 2000 分钟/月足够（每天 2 次 × 2 分钟 = 120 分钟/月） |
| Butterfly 主题更新 | 配置兼容性 | 使用 `_config.butterfly.yml` 覆盖式配置，升级时保留自定义 |
| node_modules 体积 | 构建速度 | GitHub Actions 缓存 node_modules |
| Windows SSL 证书问题 | HTTPS 请求失败 | `session.verify = False` + urllib3 警告抑制（已在 common.py 处理） |
| 抓取脚本丢失 | workflow 失败 | `blog/scripts/` 目录为空，需重建所有 Python 脚本文件 |
| 头像图片 404 | 侧边栏头像不显示 | avatar 配置指向 `/img/butterfly-icon.png`，但 `source/img/` 目录不存在 |
