# 个人静态博客网站 - 需求与架构文档


## 1. 技术栈

| 层面 | 技术 | 说明 |
|------|------|------|
| 静态站点生成器 | Hexo 8.x | Node.js 编写，Markdown 驱动，插件生态丰富，中文社区强大 |
| 主题 | Butterfly 5.5.4 | 功能最全的中文主题：内置搜索、暗色模式、卡片布局、侧边栏、TOC、代码高亮 |
| 前端搜索 | hexo-generator-searchdb | Hexo 插件，生成本地搜索索引，Butterfly 主题原生支持 |
| 内容格式 | Markdown + YAML Frontmatter | 统一写作格式，支持代码高亮、数学公式（KaTeX）、目录生成 |
| 热文抓取脚本 | Python 3 | 轻量脚本调用各平台 API，生成 JSON 数据文件 |
| 自动化构建 | GitHub Actions | 定时触发抓取脚本 + Hexo 构建 + 部署，零人工干预 |
| 部署平台 | GitHub Pages | 免费托管，支持自定义域名，与 Actions 无缝集成 |
| 版本管理 | Git + GitHub | 内容与代码统一版本控制 |
| 评论系统（可选） | Twikoo | 基于 MongoDB/Vercel，免后端，支持 Butterfly 主题集成 |

#### 抓取脚本伪代码

```python
# scripts/fetch_trending.py
import json, datetime
from sources.hacker_news import fetch_hn
from sources.github_trending import fetch_github
from sources.juejin import fetch_juejin
from sources.v2ex import fetch_v2ex
from sources.segmentfault import fetch_sf

def main():
    articles = []
    articles.extend(fetch_hn(limit=15))        # Top 15 from HN
    articles.extend(fetch_github(limit=15))     # Top 15 trending repos
    articles.extend(fetch_juejin(limit=15))     # Top 15 from 掘金
    articles.extend(fetch_v2ex(limit=10))       # Top 10 from V2EX
    articles.extend(fetch_sf(limit=10))         # Top 10 from SF

    # 按 score 排序，取 Top 50
    articles.sort(key=lambda x: x.get("score", 0), reverse=True)

    output = {
        "updated_at": datetime.datetime.utcnow().isoformat() + "Z",
        "articles": articles[:50]
    }

    with open("source/_data/trending.json", "w", encoding="utf-8") as f:
        json.dump(output, f, ensure_ascii=False, indent=2)

    print(f"Fetched {len(articles)} articles, saved top 50")

if __name__ == "__main__":
    main()
```

#### GitHub Actions 配置 (fetch_trending.yml)

```yaml
name: Fetch Trending Articles

on:
  schedule:
    - cron: "0 0,12 * * *"    # UTC 0:00 和 12:00 = 北京时间 8:00 和 20:00
  workflow_dispatch:            # 支持手动触发

jobs:
  fetch:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-python@v5
        with:
          python-version: "3.12"

      - run: pip install -r scripts/requirements.txt

      - run: python scripts/fetch_trending.py

      - name: Commit and push
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add source/_data/trending.json
          git diff --cached --quiet || git commit -m "update trending articles"
          git push
```

## 5. 快速开始命令

```bash
# 1. 安装 Node.js (>= 18.x)
# https://nodejs.org/

# 2. 安装 Hexo CLI
npm install -g hexo-cli

# 3. 创建站点
hexo init blog
cd blog

# 4. 安装 Butterfly 主题
npm install hexo-theme-butterfly --save

# 5. 安装必要插件
npm install hexo-generator-searchdb --save    # 本地搜索
npm install hexo-generator-feed --save        # RSS
npm install hexo-generator-sitemap --save     # SEO

# 6. 创建 _config.butterfly.yml（覆盖主题默认配置）

# 7. 创建第一篇文章
hexo new "我的第一篇博客"

# 8. 安装 Python 依赖（热文抓取）
pip install -r scripts/requirements.txt

# 9. 测试抓取脚本
python scripts/fetch_trending.py

# 10. 本地预览
hexo clean && hexo server

# 11. 构建部署
hexo clean && hexo generate && hexo deploy
```

## 8. 风险与注意事项

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 平台 API 变更 | 抓取失败 | 脚本模块化，单个源失败不影响其他；加错误日志 |
| 反爬限制 | 请求被拒 | 设置合理 User-Agent、请求间隔、仅抓取公开 API |
| GitHub Actions 额度超限 | 构建失败 | 免费 2000 分钟/月足够（每天 2 次 × 2 分钟 = 120 分钟/月） |
| 掘金/V2EX API 不稳定 | 数据缺失 | 多源冗余，单源故障降级处理 |
| Butterfly 主题更新 | 配置兼容性 | 使用 `_config.butterfly.yml` 覆盖式配置，升级时保留自定义 |
| node_modules 体积 | 构建速度 | GitHub Actions 缓存 node_modules |
