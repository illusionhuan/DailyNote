# 个人静态博客网站 - 需求与架构文档


## 1. 技术栈

| 层面 | 技术 | 说明 |
|------|------|------|
| 静态站点生成器 | Hexo 8.x | Node.js 编写，Markdown 驱动，插件生态丰富，中文社区强大 |
| 主题 | Butterfly 5.5.4 | 功能最全的中文主题：内置搜索、暗色模式、卡片布局、侧边栏、TOC、代码高亮 |
| 前端搜索 | hexo-generator-searchdb | Hexo 插件，生成本地搜索索引，Butterfly 主题原生支持 |
| 内容格式 | Markdown + YAML Frontmatter | 统一写作格式，支持代码高亮、数学公式（KaTeX）、目录生成 |
| 热文抓取脚本 | Python 3 | 轻量脚本调用各平台 API，生成 JSON 数据文件 |
| 自动化构建 | GitHub Actions | 定时触发抓取脚本 + Hexo 构建 + 部署，零人工干预 |
| 部署平台 | GitHub Pages | 免费托管，支持自定义域名，与 Actions 无缝集成 |
| 版本管理 | Git + GitHub | 内容与代码统一版本控制 |

#### 抓取脚本结构（✅ 已完成）

```
blog/fetch_scripts/                # 不在 scripts/ 避免 Hexo 加载为 JS
├── fetch_trending.py       # 主入口：抓取 → 排序 → 输出 JSON
├── requirements.txt        # Python 依赖：requests
└── sources/
    └── juejin.py           # 掘金 recommend_all_feed API（id_type=0）
```

**关键设计：**
- 掘金单数据源，独立 try/except 容错
- 评分归一化：`log10(raw) × 200`，映射到 0-1000 分
- SSL 兼容：Windows 禁用验证，Linux 使用系统 CA
- 输出到 `source/data/trending.json`，取 Top 30

#### GitHub Actions 配置 (fetch_trending.yml)

已创建 `.github/workflows/fetch_trending.yml`，配置如下：

```yaml
name: Fetch Trending Articles

on:
  schedule:
    - cron: "0 0,12 * * *"    # UTC 0:00 和 12:00 = 北京时间 8:00 和 20:00
  workflow_dispatch:            # 支持手动触发

permissions:
  contents: write

jobs:
  fetch:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.12"
      - name: Install dependencies
        run: pip install -r fetch_scripts/requirements.txt
        working-directory: blog
      - name: Fetch trending articles
        run: python fetch_scripts/fetch_trending.py
        working-directory: blog
      - name: Commit and push
        working-directory: blog
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add source/data/trending.json
          git diff --cached --quiet || git commit -m "update trending articles"
          git push
```

> `working-directory: blog` 确保脚本在 Hexo 项目根目录运行，`source/data/trending.json` 路径正确。

## 5. 快速开始命令

```bash
# 1. 安装 Node.js (>= 18.x)
# https://nodejs.org/

# 2. 安装 Hexo CLI
npm install -g hexo-cli

# 3. 创建站点
hexo init blog
cd blog

# 4. 安装 Butterfly 主题
npm install hexo-theme-butterfly --save

# 5. 安装必要插件
npm install hexo-generator-searchdb --save    # 本地搜索
npm install hexo-generator-feed --save        # RSS
npm install hexo-generator-sitemap --save     # SEO

# 6. 创建 _config.butterfly.yml（覆盖主题默认配置）

# 7. 创建第一篇文章
hexo new "我的第一篇博客"

# 8. 安装 Python 依赖（热文抓取）
pip install -r scripts/requirements.txt

# 9. 测试抓取脚本
python scripts/fetch_trending.py

# 10. 本地预览
hexo clean && hexo server

# 11. 构建部署
hexo clean && hexo generate && hexo deploy
```

## 9. 自定义域名绑定操作指南

> 将 `illusionhuan.github.io/DailyNote` 替换为自定义域名（如 `blog.example.com`）

### 前提条件

- 拥有一个已注册的域名（如 `example.com`）
- 博客已部署到 GitHub Pages

### 步骤 1：配置 DNS 记录

在域名注册商（阿里云、腾讯云、Cloudflare 等）的 DNS 管理面板中添加记录：

**方案 A：子域名（推荐，如 `blog.example.com`）**

| 记录类型 | 主机记录 | 记录值 | TTL |
|---------|---------|--------|-----|
| CNAME | blog | illusionhuan.github.io | 600 |

**方案 B： apex 域名（如 `example.com`）**

| 记录类型 | 主机记录 | 记录值 | TTL |
|---------|---------|--------|-----|
| A | @ | 185.199.108.153 | 600 |
| A | @ | 185.199.109.153 | 600 |
| A | @ | 185.199.110.153 | 600 |
| A | @ | 185.199.111.153 | 600 |

> apex 域名不能用 CNAME，必须用 A 记录指向 GitHub Pages 的 IP。

### 步骤 2：创建 CNAME 文件

在 `blog/source/` 下创建 `CNAME` 文件（无扩展名），内容为你的自定义域名：

```
blog.example.com
```

> 此文件会被 Hexo 复制到 `public/CNAME`，GitHub Pages 依赖它识别自定义域名。

### 步骤 3：GitHub 仓库配置

1. 进入仓库 Settings → Pages
2. 在 "Custom domain" 输入框填入 `blog.example.com`
3. 点击 Save
4. 勾选 "Enforce HTTPS"（GitHub 自动签发 Let's Encrypt 证书）

### 步骤 4：更新 Hexo 配置

修改 `blog/_config.yml`：

```yaml
url: https://blog.example.com
root: /
```

> **重要**：`root` 从 `/DailyNote/` 改为 `/`，因为自定义域名直接指向站点根目录。
> 同步更新 `_config.butterfly.yml` 中 `inject` 的 CSS/JS 路径前缀（去掉 `/DailyNote/`）。
> 同步更新 `carousel.js` 中的图片路径（去掉 `/DailyNote/` 前缀）。

### 步骤 5：验证

1. `hexo clean && hexo generate && hexo deploy`
2. 访问 `https://blog.example.com` 验证
3. 检查 HTTPS 证书是否生效（浏览器地址栏锁图标）

### 注意事项

- DNS 生效需要时间（CNAME 几分钟，A 记录可能几小时）
- GitHub Pages 的 HTTPS 证书自动签发，等待 10-15 分钟即可
- 如果使用 Cloudflare，建议关闭橙色云朵（代理），直接指向 GitHub Pages
- 修改 `root` 后必须同步更新所有硬编码路径（inject、carousel.js、CNAME 文件中的域名）
