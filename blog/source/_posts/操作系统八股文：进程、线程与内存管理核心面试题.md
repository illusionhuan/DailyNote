## 操作系统基础

### 什么是操作系统？

1. 操作系统（OS）是**管理计算机硬件与软件资源的程序**，是计算机的基石
2. 操作系统本质上是一个**运行在计算机上的软件程序**
3. 操作系统**屏蔽了硬件层的复杂性**
4. 操作系统的**内核（Kernel）**是核心部分，负责内存管理、硬件设备管理、文件系统管理和应用程序管理

### 什么是系统调用？

程序基本运行在**用户态**，如果需要调用操作系统提供的**系统态**级别子功能，就需要**系统调用**。

系统调用按功能分为：设备管理、文件管理、进程控制、进程通信、内存管理。

## 进程和线程

### 进程和线程的区别？

- **一个进程中可以有多个线程**，多个线程共享进程的堆和方法区资源
- 每个线程有自己的**程序计数器、虚拟机栈和本地方法栈**
- 线程是进程划分成的更小运行单位
- 各进程基本**独立**，各线程则**不一定**
- 线程执行**开销小**，但不利于资源管理和保护；进程正相反

### 进程有哪几种状态？

1. **创建状态（new）**：进程正在被创建
2. **就绪状态（ready）**：获得了除处理器之外的一切资源
3. **运行状态（running）**：正在处理器上运行
4. **阻塞状态（waiting）**：等待某事件而暂停运行
5. **结束状态（terminated）**：进程正在从系统中消失

### 进程间的通信方式有哪些？

1. **管道/匿名管道**：用于父子进程或兄弟进程间通信
2. **有名管道**：克服匿名管道缺点，可实现本机任意两个进程通信
3. **信号**：通知接收进程某个事件已发生
4. **消息队列**：消息的链表，存放在内核中，支持随机查询
5. **信号量**：计数器，用于多进程对共享数据的访问，主要用于同步
6. **共享内存**：多个进程访问同一块内存空间，**最有用的通信方式**
7. **套接字**：客户端和服务器之间通过网络通信

### 线程间的同步方式有哪些？

1. **互斥量（Mutex）**：互斥对象机制，如 Java 的 synchronized 和各种 Lock
2. **信号量（Semaphore）**：允许同一时刻多个线程访问同一资源，但控制最大线程数量
3. **事件（Event）**：通过通知操作保持多线程同步

### 进程的调度算法有哪些？

| 算法 | 说明 |
|------|------|
| **先到先服务（FCFS）** | 选择最先进入就绪队列的进程 |
| **短作业优先（SJF）** | 选择估计运行时间最短的进程 |
| **时间片轮转（RR）** | 每个进程分配一个时间片，最古老、最公平、使用最广 |
| **多级反馈队列** | 既能使高优先级作业得到响应，又能使短作业迅速完成（UNIX 采用） |
| **优先级调度** | 首先执行最高优先级的进程 |

## 死锁

### 什么是死锁？

多个进程/线程同时被阻塞，它们中的一个或全部都在等待某个资源被释放，程序不可能正常终止。

### 死锁的四个必要条件

1. **互斥**：资源一次只有一个进程可以使用
2. **占有并等待**：进程至少占有一个资源，并等待另一个资源
3. **非抢占**：资源不能被抢占
4. **循环等待**：存在等待资源的循环链

### 如何解决死锁？

**预防**（破坏四个条件之一）：
- **静态分配策略**：进程执行前申请到所有资源（破坏占有并等待）
- **层次分配策略**：资源分层，只能申请较高层资源（破坏循环等待）

**避免**：银行家算法——先试探分配资源，通过安全性算法判断分配后系统是否安全

**检测**：通过进程-资源分配图检测，类似于拓扑排序

**解除**：
1. 立即结束所有进程重新启动
2. 撤销涉及死锁的所有进程
3. 逐个撤销进程回收资源
4. 抢占资源

## 内存管理

### 常见的内存管理机制

1. **块式管理**：将内存分为固定大小的块，浪费严重
2. **页式管理**：把主存分为大小相等且固定的页，提高利用率，减少碎片
3. **段式管理**：把主存分为有实际意义的段（主程序段、数据段等）
4. **段页式管理**：先分段，每段再分页，结合两者优点

> 页是**物理单位**，段是**逻辑单位**。分页提高内存利用率，分段更好满足用户需求。

### 快表和多级页表

**快表（TLB）**：特殊的高速缓冲存储器，作为页表的 Cache，加速虚拟地址到物理地址的转换。

**多级页表**：避免把全部页表一直放在内存中占用过多空间（时间换空间的典型场景）。

### 分页和分段的异同？

**共同点**：都是为了提高内存利用率，减少内存碎片，都是离散分配内存。

**区别**：
- 页的大小**固定**，由操作系统决定；段的大小**不固定**
- 分页满足**内存管理**需求；分段是**逻辑信息**单位

### 为什么需要虚拟地址空间？

没有虚拟地址空间时，程序直接访问物理内存：
1. 用户程序可以**任意访问内存**，容易破坏操作系统
2. **无法同时运行多个程序**（内存地址互相覆盖）

虚拟地址的优势：访问不相邻的物理内存、访问大于物理内存的空间、进程间地址隔离。

## 虚拟内存

### 什么是虚拟内存？

虚拟内存让程序可以拥有**超过系统物理内存大小**的可用内存空间。通过将内存扩展到硬盘空间，为每个进程提供一致的、私有的地址空间。

### 局部性原理

1. **时间局部性**：某条指令执行后不久可能再次执行（循环操作）
2. **空间局部性**：访问某个存储单元后，其附近的存储单元也将被访问（顺序执行）

### 虚拟内存的实现方式

1. **请求分页存储管理**：最常用，增加请求调页和页面置换功能
2. **请求分段存储管理**：增加请求调段和分段置换功能
3. **请求段页式存储管理**：结合两者

### 页面置换算法有哪些？

| 算法 | 说明 |
|------|------|
| **OPT** | 淘汰以后永不使用或最长时间不使用的页面（无法实现，仅作衡量标准） |
| **FIFO** | 淘汰最先进入内存的页面 |
| **LRU** | 淘汰最近最久未使用的页面 |
| **LFU** | 淘汰之前时期使用最少的页面 |
