## 前言

Claude Code 是 Anthropic 推出的 CLI 编程助手，直接在终端中运行，能读写文件、执行命令、管理 Git 工作流。本文整理 Claude Code 中常用的斜杠命令（Slash Commands）及其作用，帮助快速上手。

<!-- more -->

## 基础操作命令

### /help

显示帮助信息，列出所有可用命令和快捷键。遇到任何疑问时的第一个入口。

### /clear

清空当前对话上下文。当对话过长、上下文混乱或切换任务时使用，等价于重新开始一个全新会话。

### /compact

压缩当前对话上下文。与 `/clear` 不同，它会保留关键信息的摘要而非直接丢弃。适合对话变长但还需要保留上下文的场景。

### /cost

显示当前会话的 token 用量和费用统计。包括输入 token、输出 token 和缓存命中的详细数据，方便控制成本。

### /doctor

检查 Claude Code 的安装状态和环境配置是否正常。遇到问题时先跑一遍 `/doctor` 排查基础环境。

## 模型与性能

### /model

切换使用的模型。可以在 Opus、Sonnet、Haiku 之间切换，不同模型在能力和速度之间有不同的取舍：

| 模型 | 特点 | 适用场景 |
|------|------|---------|
| Opus | 最强推理能力 | 复杂架构设计、多文件重构 |
| Sonnet | 平衡速度与能力 | 日常开发、代码审查 |
| Haiku | 最快响应 | 简单问答、快速查询 |

### /fast

切换快速模式。快速模式使用更快的输出通道但不降级模型能力，适合需要快速迭代的场景。

## 配置与权限

### /config

打开配置管理。可调整主题、默认模型、API Key 等全局设置。

### /permissions

查看和管理当前的工具权限。Claude Code 的每个工具调用（文件读写、命令执行等）都需要权限授权，这个命令可以查看当前权限状态并进行调整。

## 代码审查与工作流

### /review

对当前分支的 PR 进行代码审查。Claude Code 会分析 diff，找出潜在的 bug、安全问题和风格问题，并以评论形式反馈。

### /init

在当前项目根目录初始化 `CLAUDE.md` 文件。这个文件是 Claude Code 的项目级指引，写明项目结构、常用命令、技术栈和编码规范，让 Claude Code 在后续会话中更好地理解项目。

### /memory

打开记忆文件编辑。Claude Code 有持久化的记忆系统，存储在 `~/.claude/projects/` 下，通过 `/memory` 可以直接编辑这些跨会话保留的信息。

## 终端与编辑

### /terminal-setup

配置终端集成。设置终端以获得更好的交互体验，比如支持鼠标事件、正确的颜色渲染等。

### /vim

切换 Vim 编辑模式。开启后在输入框中可以使用 Vim 的快捷键进行文本编辑，适合 Vim 用户。

## 核心工具能力

除了斜杠命令，Claude Code 的核心价值在于内置的工具能力：

### 文件操作

- **Read** — 读取文件内容，支持文本和图片
- **Write** — 创建或覆盖文件
- **Edit** — 精确的字符串替换编辑，只修改需要变更的部分
- **Glob** — 按模式匹配查找文件
- **Grep** — 基于 ripgrep 的内容搜索

### 命令执行

- **Bash** — 执行 shell 命令，用于运行构建、测试、安装依赖等操作

### 子代理（Agent）

Claude Code 可以派生子代理并行处理任务。不同类型的子代理各有专长：

- **Explore** — 快速只读搜索，定位代码位置
- **Plan** — 架构设计，制定实施方案
- **Claude** — 通用型，处理各类复杂任务

子代理的价值在于：并行执行互不依赖的搜索任务，保护主对话的上下文窗口不被大量搜索结果淹没。

### 任务管理

- **TaskCreate / TaskUpdate / TaskList** — 创建和跟踪任务清单，适合多步骤的复杂工作
- **EnterPlanMode** — 进入规划模式，先设计实施方案再动手写代码

## 快捷键

| 快捷键 | 作用 |
|--------|------|
| `Esc` | 中断当前生成 |
| `Ctrl+C` | 取消当前输入 |
| `Up/Down` | 浏览历史输入 |
| `Tab` | 自动补全 |
| `!command` | 直接在会话中执行 shell 命令 |

## 实际使用技巧

### 1. 项目初始化流程

```bash
# 进入项目目录
cd my-project

# 启动 Claude Code
claude

# 初始化项目指引
/init
```

### 2. 切换模型应对不同任务

```
/model opus        # 复杂重构任务
/model sonnet      # 日常开发
/model haiku       # 快速查询
```

### 3. 长对话管理

```
/compact           # 对话变长时压缩上下文
/clear             # 切换任务时清空重来
```

### 4. 权限管理

```
/permissions       # 查看当前权限
# 在 settings.json 中预配置常用权限减少弹窗
```

## 总结

Claude Code 的命令体系围绕三个核心设计：**交互控制**（clear/compact/cost）、**能力配置**（model/config/permissions）、**工程集成**（review/init/memory）。掌握这些命令能显著提升与 AI 协作编程的效率。
