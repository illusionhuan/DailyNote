## 面试求职指南

### 尽早以求职为导向来学习

建议大学生尽早以求职为导向学习，更有针对性，减少迷茫，少走弯路。但**不要理解为不用学计算机基础课程**！操作系统、计算机组成原理、计算机网络在面试和工作中都会用到。

**如何做**：根据招聘岗位要求梳理技能清单 → 按清单学习和提升 → 写好简历。这是**以终为始**思想的运用。

### 了解投递简历的黄金时间

- **秋招**：7 月开始，持续到 9 月底。大厂 9 月中旬可能就没 HC 了
- **春招**：3 月开始，持续到 4 月底。大厂 3 月中旬可能就没 HC 了

### 获取招聘信息的渠道

1. **目标企业官网/公众号**：最及时最权威
2. **牛客网**：大量招聘信息和内推帖子
3. **超级简历**：整合各大企业校园招聘入口
4. **认识的朋友**：了解信息并帮忙内推
5. **宣讲会**：直接去目标学校参加
6. **校园就业网站、招聘网站**

### 多花时间完善简历

**至少花 2~3 天专门完善简历。** 简历是门面，HR 可能只花 10 秒就决定 Pass 还是 Fail。

### 提前准备技术面试和手撕算法

面试涉及的知识点方向：

- **Java**：基础、集合、并发、JVM
- **计算机基础**：算法、数据结构、网络、操作系统
- **数据库**：MySQL、Redis
- **常用框架**：Spring、Spring Boot、MyBatis
- **分布式**：CAP/BASE 理论、RPC、分布式事务、分布式 ID
- **高并发**：消息队列、读写分离、分库分表、负载均衡
- **高可用**：限流、降级、熔断

### 提前准备自我介绍

好的自我介绍应该：
1. 重点放在自己在行的地方和优势
2. 突出自己的能力（如定位 bug 能力特别厉害）
3. 社招侧重项目经验和技术能力，校招侧重学习能力和潜力

### 面试之后及时复盘

面试失败不要灰心，通过不要狂喜。面试就像全新征程，失败和胜利都是平常之事。

## 简历撰写指南

### 简历内容

| 模块 | 要点 |
|------|------|
| **个人信息** | 姓名、年龄、电话、邮箱；加分项：Github、技术博客 |
| **求职意向** | 应聘岗位、期望城市 |
| **教育经历** | 学历、专业、毕业学校、毕业日期 |
| **专业技能** | 技术名词规范大小写（Java 不是 java），突出亮点 |
| **实习/工作经历** | 时间倒序，最有价值的放最前面 |
| **项目经历** | 精选 2~3 个，突出亮点，成果量化 |
| **荣誉奖项** | 高含金量竞赛获奖写上，适当往前放 |
| **个人评价** | 突出亮点，用具体事例证明 |

### STAR 法则

- **Situation**：事情是在什么情况下发生的？
- **Task**：你的任务是什么？
- **Action**：你做了什么？
- **Result**：最终结果怎样？

### FAB 法则

- **Feature**：你的特征/优势是什么？
- **Advantage**：比别人好在哪些地方？
- **Benefit**：如果雇佣你，招聘方会得到什么好处？

### 注意事项

1. **一定使用 PDF 格式投递**
2. 不会的东西不要写在简历上
3. 项目经历不在于多，在于有亮点
4. 技术名词规范大小写：MySQL、Java、Spring Boot
5. 简历最后加一句："感谢您花时间阅读我的简历，期待能有机会和您共事。"

## 面试常见词汇扫盲

| 词汇 | 解释 |
|------|------|
| **春招/秋招** | 春招 3~4 月（实习+补招），秋招 7~9 月（应届生关键时期） |
| **暑期实习/日常实习** | 暑期实习春招招聘，6~7 月入职；日常实习全年进行，部门散招 |
| **提前批** | 各公司提前抢夺优秀毕业生，不影响秋招，很多免笔试 |
| **内推** | 有用内推：内推者直接把简历交到 HR 手里 |
| **HC** | Headcount，公司打算招的人数 |
| **面试记录** | 面试官评语，后续面试官和 HR 都能看到 |
| **预面试** | 不录入公司系统的面试，不过不影响投其他部门 |
| **主管面** | 技术主管面试，半技术半聊天，考察团队契合度 |
| **HR 面** | 人力资源面试，考察性格、稳定性、薪资期望 |
| **八股文** | 概念性面试题，有固定套路 |
| **手撕算法** | 面试官出算法题现场 coding |

## 项目经验指南

### 没有项目经验怎么办？

1. **实战项目视频/专栏**：跟着老师一起做，要有自己的思考和改进
2. **实战类开源项目**：理解代码后进行改进或增加功能
3. **从头开始做**：遇到不会的临时学，现学现卖
4. **参加大公司组织的大赛**：获奖含金量很高
5. **参与实际项目**：老师项目、私活、实习项目

### 跟着视频做的项目会被嫌弃吗？

大部分面试官能理解应届生没有实际项目经验。关键是**自己要有改进和优化**，不要浅尝辄止。

**容易优化的点**：
1. 全局异常处理
2. 技术选型优化（如 Guava 换 Caffeine）
3. 数据库优化（索引、SQL、读写分离）
4. 引入缓存提高响应速度
5. 安全问题排查

### 项目经历优化建议

从**性能优化方向**入手性价比更高，更容易体现在简历上，面试官也更喜欢提问。成果要量化：**"使用 xxx 技术解决了 xxx 问题，系统 QPS 从 xxx 提高到了 xxx"**。

### 必须是微服务项目才有亮点？

**不是。** 单体项目做到极致也很好。微服务涉及知识点太多，一般人很难吃透，面试官也会质疑是否独立完成。重要的是让项目有**亮点**（如何提升性能、如何解决痛点）。
