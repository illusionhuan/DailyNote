## MySQL 基础架构

### MySQL 的架构是怎样的？

MySQL 主要由以下部分构成：

1. **连接器**：身份认证和权限验证
2. **查询缓存**：执行查询时先查缓存（**MySQL 8.0 已移除**）
3. **分析器**：检查 SQL 语句要干嘛，语法是否正确
4. **优化器**：按 MySQL 认为最优的方案执行
5. **执行器**：执行语句，判断权限后从存储引擎返回数据
6. **插件式存储引擎**：负责数据的存储和读取（InnoDB、MyISAM 等）

## MySQL 存储引擎

### MyISAM 和 InnoDB 的区别是什么？

| 对比项 | MyISAM | InnoDB |
|--------|--------|--------|
| **行级锁** | 仅表级锁 | 支持行级锁和表级锁 |
| **事务** | 不支持 | 支持（四种隔离级别） |
| **外键** | 不支持 | 支持 |
| **崩溃恢复** | 不支持 | 支持（依赖 redo log） |
| **MVCC** | 不支持 | 支持 |
| **索引实现** | 索引和数据文件分离 | 数据文件本身就是索引文件（聚簇索引） |

**选择建议**：一般情况下选择 InnoDB。MySQL 5.5.5 之后 InnoDB 已是默认存储引擎。

## MySQL 事务

### 何谓事务？

事务是逻辑上的**一组操作，要么都执行，要么都不执行**。经典例子是转账：小明给小红转账 1000 元，两个操作必须都成功或都失败。

### ACID 特性

- **原子性（Atomicity）**：事务是最小执行单位，不允许分割
- **一致性（Consistency）**：执行事务前后，数据保持一致
- **隔离性（Isolation）**：并发事务之间互不干扰
- **持久性（Durability）**：事务提交后，改变是持久的

> **A、I、D 是手段，C 是目的**——只有保证了原子性、隔离性、持久性，一致性才能得到保障。

### 并发事务带来哪些问题？

| 问题 | 说明 |
|------|------|
| **脏读** | 读取到未提交的数据 |
| **丢失修改** | 两个事务同时修改同一数据，一个的修改被覆盖 |
| **不可重复读** | 同一事务内两次读取同一数据，结果不同（被其他事务修改） |
| **幻读** | 同一事务内两次查询，第二次多了原本不存在的记录 |

> **不可重复读**重点是记录修改/减少；**幻读**重点是记录新增。

### SQL 标准定义了哪些事务隔离级别？

| 隔离级别 | 脏读 | 不可重复读 | 幻读 |
|----------|------|-----------|------|
| READ-UNCOMMITTED（读未提交） | 有 | 有 | 有 |
| READ-COMMITTED（读已提交） | 无 | 有 | 有 |
| **REPEATABLE-READ（可重复读）** | 无 | 无 | 有 |
| SERIALIZABLE（可串行化） | 无 | 无 | 无 |

**MySQL 默认隔离级别**：**REPEATABLE-READ**（可重复读）。

### MySQL 的隔离级别是基于锁实现的吗？

基于**锁和 MVCC** 机制共同实现。**SERIALIZABLE** 通过锁实现，其他级别基于 **MVCC** 实现。REPEATABLE-READ 在当前读情况下需要使用加锁读来保证不出现幻读。

## MySQL 锁

### 表级锁和行级锁

| 对比项 | 表级锁 | 行级锁 |
|--------|--------|--------|
| 粒度 | 最大 | 最小 |
| 加锁 | 快 | 慢 |
| 冲突概率 | 高 | 低 |
| 并发度 | 低 | 高 |
| 死锁 | 不会 | 会 |

**注意事项**：InnoDB 的行锁是针对**索引字段**加的锁。如果 WHERE 条件没有命中索引，会导致**扫描全表对所有行记录加锁**！

### 共享锁和排他锁

- **共享锁（S 锁/读锁）**：允许多个事务同时获取
- **排他锁（X 锁/写锁）**：不允许多个事务同时获取

| | S 锁 | X 锁 |
|--|------|------|
| S 锁 | 不冲突 | **冲突** |
| X 锁 | **冲突** | **冲突** |

### 意向锁有什么作用？

意向锁是**表级锁**，用于快速判断表中是否有行锁，避免一行行遍历：

- **意向共享锁（IS）**：事务有意向对某些行加 S 锁
- **意向排他锁（IX）**：事务有意向对某些行加 X 锁

意向锁之间**互相兼容**，但与表级 S 锁和 X 锁互斥。

### InnoDB 有哪几类行锁？

| 锁类型 | 说明 |
|--------|------|
| **记录锁（Record Lock）** | 锁定单个行记录 |
| **间隙锁（Gap Lock）** | 锁定一个范围，**不包括**记录本身 |
| **临键锁（Next-Key Lock）** | Record Lock + Gap Lock，锁定范围**包含**记录本身 |

### 当前读和快照读有什么区别？

**快照读**：单纯的 SELECT 语句，读取行的历史版本（MVCC）
- RC 级别：总是读取**最新快照**
- RR 级别：总是读取**事务开始时的行数据版本**

**当前读**：给行记录加 X 锁或 S 锁
```sql
SELECT ... FOR UPDATE        -- 加X锁
SELECT ... LOCK IN SHARE MODE -- 加S锁
INSERT / UPDATE / DELETE       -- 加X锁
```
