## 前言

JVM 是 Java 程序的运行基石。理解 JVM 内存模型、类加载机制和垃圾回收算法，不仅能帮你排查 OOM、StackOverflow 等运行时问题，更是深入理解 Java 并发、性能调优的前提。本文系统梳理 JVM 的核心知识体系。

## 一、JVM 运行时数据区域

```
┌─────────────────────────────────────────────┐
│                  JVM 进程                     │
│                                              │
│  ┌──────────────┐  ┌───────────────────────┐ │
│  │ 程序计数器    │  │     堆（Heap）         │ │
│  │ （线程私有）   │  │   （线程共享）         │ │
│  └──────────────┘  │                       │ │
│  ┌──────────────┐  │  对象实例、数组        │ │
│  │ 虚拟机栈      │  │                       │ │
│  │ （线程私有）   │  └───────────────────────┘ │
│  │              │  ┌───────────────────────┐ │
│  │ 栈帧×N       │  │   方法区/元空间         │ │
│  └──────────────┘  │   （线程共享）         │ │
│  ┌──────────────┐  │                       │ │
│  │ 本地方法栈    │  │  类信息、常量池、       │ │
│  │ （线程私有）   │  │  静态变量             │ │
│  └──────────────┘  └───────────────────────┘ │
└─────────────────────────────────────────────┘
```

### 1.1 程序计数器

- 记录当前线程执行的字节码行号
- 线程私有，每个线程一个
- 唯一不会产生 `OutOfMemoryError` 的区域

### 1.2 虚拟机栈

每个方法调用时创建一个**栈帧**，包含：

| 组件 | 作用 |
|------|------|
| 局部变量表 | 存放方法参数和局部变量（基本类型 + 引用地址） |
| 操作数栈 | 方法执行时的临时数据栈 |
| 动态链接 | 指向运行时常量池的引用 |
| 方法返回地址 | 方法退出后回到的位置 |

**两种异常：**
- `StackOverflowError`：栈深度超过限制（如无限递归）
- `OutOfMemoryError`：栈扩展时内存不足

### 1.3 本地方法栈

为 `native` 方法服务（如 `Thread.start0()`），与虚拟机栈类似。

### 1.4 堆

- **最大的一块内存区域**，所有对象实例都在堆上分配
- 线程共享
- GC（垃圾回收）的主要工作区域
- 分代结构：新生代（Eden + S0 + S1）+ 老年代

```
堆
├── 新生代（Young Generation）
│   ├── Eden（伊甸区）    — 对象首次分配
│   ├── Survivor 0（S0） — 存活对象
│   └── Survivor 1（S1） — 存活对象
└── 老年代（Old Generation） — 长期存活的对象
```

### 1.5 方法区

存储类信息、常量池、静态变量、JIT 编译后的代码。

- JDK 7 及之前：永久代（PermGen），在堆中
- JDK 8+：元空间（Metaspace），使用本地内存

## 二、对象的创建过程

### 2.1 创建步骤

1. **类加载检查**：检查类是否已加载、解析、初始化
2. **分配内存**：指针碰撞（内存规整时）或空闲列表（内存不规整时）
3. **初始化零值**：将分配的内存空间初始化为零值
4. **设置对象头**：Mark Word（哈希码、GC 分代年龄、锁标志）+ 类型指针
5. **执行 `<init>`**：执行构造方法

### 2.2 对象内存布局

```
对象头（Header）
├── Mark Word（8 字节）：哈希码、GC 年龄、锁状态
└── 类型指针（4 字节）：指向方法区的类元数据

实例数据（Instance Data）
└── 字段的实际值

对齐填充（Padding）
└── 对齐到 8 字节的整数倍
```

## 三、类加载机制

### 3.1 类的生命周期

```
加载 → 验证 → 准备 → 解析 → 初始化 → 使用 → 卸载
```

| 阶段 | 作用 |
|------|------|
| 加载 | 通过全限定名获取类的字节码流，存入方法区 |
| 验证 | 检查字节码的正确性和安全性 |
| 准备 | 为静态变量分配内存并设置零值（final 常量直接赋值） |
| 解析 | 将符号引用替换为直接引用 |
| 初始化 | 执行 `<clinit>` 方法（静态变量赋值 + 静态代码块） |

### 3.2 三种类加载器

| 加载器 | 加载范围 |
|--------|----------|
| Bootstrap ClassLoader | `JAVA_HOME/lib`（rt.jar 等核心类库） |
| Extension ClassLoader | `JAVA_HOME/lib/ext`（扩展类库） |
| Application ClassLoader | `classpath`（应用代码） |

### 3.3 双亲委派模型

```
Application ClassLoader
  ↓ 委派
Extension ClassLoader
  ↓ 委派
Bootstrap ClassLoader
  ↓ 尝试加载
  ↓ 失败则回退
Extension ClassLoader 尝试加载
  ↓ 失败则回退
Application ClassLoader 加载
```

**工作流程：** 子加载器收到加载请求时，先委托父加载器加载。只有父加载器无法完成时，子加载器才自己加载。

**意义：**
1. 避免类的重复加载
2. 保护核心类库的安全（防止自定义 `java.lang.String` 替换核心类）

**打破双亲委派：** 重写 `loadClass()` 方法。典型场景：SPI（Service Provider Interface）、Tomcat 类加载、OSGi。

## 四、垃圾回收

### 4.1 对象存活判定

**可达性分析算法：** 从 GC Roots 出发，沿引用链遍历，不可达的对象即为垃圾。

**GC Roots 包括：**
- 虚拟机栈中引用的对象
- 方法区中静态属性引用的对象
- 方法区中常量引用的对象
- 本地方法栈中引用的对象

### 4.2 四种引用类型

| 引用类型 | 回收时机 | 用途 |
|----------|----------|------|
| 强引用 | 不回收（`Object o = new Object()`） | 普通使用 |
| 软引用 | 内存不足时回收 | 缓存 |
| 弱引用 | 下次 GC 时回收 | WeakHashMap |
| 虚引用 | 随时回收，无法获取对象 | 跟踪回收状态 |

### 4.3 GC 算法

| 算法 | 原理 | 优缺点 |
|------|------|--------|
| 标记-清除 | 标记存活对象，清除未标记 | 简单，产生内存碎片 |
| 标记-整理 | 标记后将存活对象向一端移动 | 无碎片，移动开销大 |
| 复制算法 | 将存活对象复制到另一块空间 | 无碎片，空间利用率 50% |
| 分代收集 | 新生代用复制算法，老年代用标记-整理 | 主流方案 |

### 4.4 常见垃圾收集器

| 收集器 | 区域 | 算法 | 特点 |
|--------|------|------|------|
| Serial | 新生代 | 复制 | 单线程，STW |
| ParNew | 新生代 | 复制 | 多线程版 Serial |
| Parallel Scavenge | 新生代 | 复制 | 关注吞吐量 |
| CMS | 老年代 | 标记-清除 | 低停顿，有碎片 |
| G1 | 全堆 | 分区 | 可预测停顿时间 |
| ZGC | 全堆 | 着色指针 | 亚毫秒停顿 |

### 4.5 Minor GC vs Full GC

| 类型 | 触发条件 | 停顿时间 |
|------|----------|----------|
| Minor GC | Eden 区满 | 短（毫秒级） |
| Full GC | 老年代满 / 方法区满 / System.gc() | 长（可能秒级） |

**对象晋升规则：**
- 对象在 Eden 出生，经过 Minor GC 存活后进入 Survivor
- 每次 Minor GC 存活，年龄 +1
- 年龄达到阈值（默认 15）晋升到老年代
- 大对象直接进入老年代

## 五、JVM 调优常用参数

```bash
# 堆内存
-Xms512m          # 初始堆大小
-Xmx1024m         # 最大堆大小
-Xmn256m          # 新生代大小

# 元空间
-XX:MetaspaceSize=128m
-XX:MaxMetaspaceSize=256m

# GC 日志
-XX:+PrintGCDetails
-XX:+PrintGCDateStamps
-Xloggc:gc.log

# 收集器选择
-XX:+UseG1GC
-XX:MaxGCPauseMillis=200
```

## 总结

| 知识点 | 核心要点 |
|--------|----------|
| 内存区域 | 堆（对象）、栈（方法调用）、方法区（类信息）、程序计数器 |
| 对象创建 | 类加载→分配内存→零值初始化→设置对象头→构造方法 |
| 类加载 | 双亲委派模型：先委托父加载器，保护核心类安全 |
| GC 算法 | 分代收集：新生代复制 + 老年代标记-整理 |
| 收集器 | G1 是主流选择，ZGC 是未来趋势 |

JVM 是 Java 面试的重灾区，理解内存模型和 GC 机制是性能调优的基础。建议结合 `jps`、`jstat`、`jmap`、`jvisualvm` 等工具进行实操练习。
